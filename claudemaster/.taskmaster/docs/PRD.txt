# 모바일 티켓예약 시스템 웹 PRD (UI 구현 중심)

## Overview

모바일 환경에서 티켓을 검색하고 좌석을 선택하여 예약할 수 있는 UI 구현. 영화, 공연 등의 이벤트 티켓을 간편하게 예매하는 인터페이스에 집중.

---

## Core Features

### 1. 이벤트 탐색
- 카테고리별 이벤트 카드 리스트
- 포스터 이미지, 제목, 기본 정보 표시

### 2. 이벤트 상세
- 포스터, 설명, 장소 정보
- 날짜/시간 선택 UI
- "좌석 선택" 버튼

### 3. 좌석 선택
- 좌석 배치도 (클릭/터치로 선택)
- 선택한 좌석 및 금액 표시

### 4. 예약 확인
- 선택 정보 요약
- 사용자 정보 입력 폼
- QR 티켓 표시

### 5. 예약 내역
- 예약 리스트 카드
- QR 코드 재확인

---

## User Experience

### User Flow
1. 메인 → 이벤트 선택 → 상세 페이지
2. 날짜/시간 선택 → 좌석 선택
3. 정보 입력 → 예약 완료 → QR 티켓

### UI/UX 원칙
- 모바일 우선 (터치 최적화)
- 하단 고정 CTA 버튼
- 명확한 단계별 진행 표시
- 선택 항목 시각적 피드백

## Development Roadmap

### Phase 1: 레이아웃 & 이벤트 리스트
- 프로젝트 세팅 (React + Tailwind)
- 헤더, 하단 네비게이션
- 메인 페이지: 이벤트 카드 리스트
- Mock 데이터 (JSON)

### Phase 2: 이벤트 상세 & 시간 선택
- 상세 페이지 UI
- 날짜/시간 선택 컴포넌트
- 선택 상태 관리

### Phase 3: 좌석 선택
- 좌석 맵 UI (그리드)
- 좌석 클릭 인터랙션
- 하단 선택 정보 바 (좌석, 금액)

### Phase 4: 예약 확인
- 예약 정보 요약
- 사용자 정보 폼
- 예약 완료 페이지 (QR 코드)
- localStorage 저장

### Phase 5: 예약 내역 & 검색
- 마이페이지 (예약 리스트)
- 검색 UI
- 필터 UI

### Phase 6: UI 완성도
- 로딩 인디케이터
- 토스트 알림
- 페이지 전환 애니메이션
- 에러 페이지

---

## Logical Dependency Chain

1. **Phase 1** → 기본 구조 (다른 모든 페이지의 기반)
2. **Phase 2** → 상세 정보 (예약 플로우 시작)
3. **Phase 3** → 핵심 기능 (좌석 선택)
4. **Phase 4** → 플로우 완성 (예약 완료)
5. **Phase 5** → 부가 기능 (검색, 내역)
6. **Phase 6** → 완성도 (UX 개선)

각 단계는 독립적으로 완성 가능하며, 이전 단계 위에 쌓아갈 수 있음.

---

## Risks and Mitigations

### 좌석 맵 복잡도
- **Risk**: 작은 화면에서 터치 정확도
- **Solution**: 충분한 터치 영역, 줌 기능

### 기능 과욕
- **Risk**: 불필요한 기능 추가로 일정 지연
- **Solution**: Phase 1-4를 절대 MVP로 고정

### 디자인 품질
- **Risk**: 디자이너 없이 UI 완성도 부족
- **Solution**: 기존 앱 벤치마킹, Tailwind 활용
